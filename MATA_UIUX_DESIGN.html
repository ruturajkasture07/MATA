Foundation

Design Principles

Six core beliefs that drive every pixel, every animation, and every interaction in MATA.

01
👁️
Blind-First Design

Every screen must work perfectly without looking at it. If a visually impaired user can't navigate purely by touch and sound, the design has failed.

02
🎯
One Action Per Screen

Each screen has one massive, unmissable primary action. No cognitive overload. No hunting for buttons. Tap anywhere to hear what to do.

03
✨
Premium by Default

Students deserve beautiful tools. Deep space backgrounds, vibrant gradients, glassmorphism, and fluid animations make learning feel exciting — not clinical.

04
⚡
Instant Feedback

Every action gets immediate visual + haptic + audio response within 100ms. Users should never wonder "did that work?" — they should know.

05
🔊
Voice as Primary Input

Voice commands, Q&A, and navigation are first-class features — not afterthoughts. The entire app should be operable without ever typing.

06
🧒
Age-Adaptive UI

The interface subtly adapts to the selected learning mode. Child mode: rounder, warmer, more emoji. Adult mode: crisper, more information-dense.

Interaction Design

Gesture Language

A complete, consistent gesture vocabulary that works for both sighted and visually impaired users — no visual memory required.

☝️
Single Tap

Reads the element label aloud via NarratorService. Explores without activating.

👆👆
Double Tap

Activates the selected element — opens screens, presses buttons, confirms actions.

✋
Long Press

Hold to talk — starts voice recording in Q&A chat. Release to send the voice input.

👋
Swipe Left/Right

Navigate between bottom tabs. Swipe left = next tab; swipe right = previous tab.

☝️☝️
Two-Finger Swipe Up

Trigger camera scan from any screen. Global shortcut — no navigation required.

🤙
Shake Phone

Invoke MARK from anywhere. Shake = "Hey MARK!" — no need to find the FAB button.

🤏
Pinch In

Decrease TTS speech speed (→ 0.3x). Useful for complex technical explanations.

🤙🤙
Pinch Out

Increase TTS speech speed (→ 1.5x or 2x). Useful for review or familiar content.

Accessibility

Accessibility First

MATA is built for the 285 million people worldwide with visual impairment. Accessibility is not a feature — it is the product.

👁️
Vision Accessibility
  • Built-in screen reader (NarratorService) — no OS screen reader dependency
  • Every element has a descriptive Semantics label
  • Minimum font size: 16px; primary content: 24–40px
  • WCAG AA contrast ratios on all text-background pairs
  • High-contrast mode toggle (fully functional in new design)
  • No information conveyed by color alone (icons + text always paired)
  • Camera guidance audio: "Tilt left", "Move closer", "Hold still"
🔊
Audio Accessibility
  • All content narrated via TTS at adjustable speed (0.25×–2×)
  • Screen-by-screen context announcements on navigation
  • Earcons: distinct sounds for capture success, load complete, error
  • MARK bot voice Q&A — no text typing required ever
  • Wake word support: "Hey MARK" (phone shake as fallback)
  • Audio playback with voice commands: "pause", "repeat", "faster"
  • Braille display compatibility via accessibility services bridge
🤲
Motor Accessibility
  • Minimum touch target: 72px × 72px (exceeds 44px WCAG minimum)
  • Generous spacing between interactive elements (≥16px)
  • Hold-to-capture: 3-second steady hold triggers auto-capture
  • Voice-complete mode: zero touch needed after app is open
  • Switch Access / AssistiveTouch compatibility via Semantics
  • No time-limited interactions (no disappearing toasts without narration)
🧠
Cognitive Accessibility
  • One primary action per screen — no choice paralysis
  • Age-adaptive explanations: Child / Teen / Adult modes
  • Comprehension check after each explanation
  • If marked "Too Hard" → automatically simplifies language
  • Consistent navigation patterns across all screens
  • Plain language in all UI copy — no jargon
  • Tutorial mode on first launch with full narrated walkthrough
Developer Guide

How to Implement This Design

A step-by-step Flutter implementation guide for bringing this design system to life in the MATA codebase.

1
Create app_theme.dart — Design Tokens

Extract all colors, typography, and spacing into a single source of truth. Never hardcode colors in widget files again.

// lib/theme/app_theme.dart class AppColors { static const bg0 = Color(0xFF060611); static const bg1 = Color(0xFF0D0D1A); static const primary = Color(0xFF7C3AED); static const primaryLight = Color(0xFFA855F7); static const accent = Color(0xFF06B6D4); static const accent2 = Color(0xFF10B981); static const danger = Color(0xFFEF4444); static const textPrimary = Color(0xFFF8FAFC); static const textSecondary = Color(0xFF94A3B8); static const gradientPrimary = LinearGradient( colors: [Color(0xFF7C3AED), Color(0xFFA855F7), Color(0xFF06B6D4)], begin: Alignment.topLeft, end: Alignment.bottomRight, ); }
2
Create glass_card.dart — Reusable Widget

Replace every repeated BackdropFilter + Container pattern with a single reusable GlassCard widget.

// lib/widgets/glass_card.dart class GlassCard extends StatelessWidget { final Widget child; final double blur; final double opacity; final BorderRadius? radius; const GlassCard({required this.child, ...}); @override Widget build(BuildContext context) { return ClipRRect( borderRadius: radius ?? BorderRadius.circular(24), child: BackdropFilter( filter: ImageFilter.blur(sigmaX: blur, sigmaY: blur), child: Container( decoration: BoxDecoration( color: Colors.white.withOpacity(opacity), borderRadius: radius ?? BorderRadius.circular(24), border: Border.all(color: Colors.white.withOpacity(0.08)), ), child: child, ), ), ); } }
3
Upgrade NarratorService — Rich Audio Cues

Add earcon sounds, queuing system, and voice command recognition to make the narrator truly first-class.

// lib/services/narrator_service.dart class NarratorService { // Add audio cue support Future<void> playEarcon(Earcon type) async { ... } // Add narration queue to prevent interruptions final Queue<String> _queue = Queue(); // Add speed control Future<void> setSpeed(double rate) async { ... } // Enum for earcons enum Earcon { success, error, navigate, capture, loading } }
4
Implement Real History Screen

Add GET /api/v1/sessions endpoint to backend, create SessionCard widget, add filter chips by subject, and enable replay via session_id.

// Backend: main.py — add new endpoint @app.get("/api/v1/sessions") def get_sessions(user_id: str, db: Session = Depends(get_db)): sessions = db.query(SessionLog)\ .order_by(SessionLog.timestamp.desc())\ .limit(20).all() return {"sessions": [s.__dict__ for s in sessions]}
5
Wire Settings with SharedPreferences

Create a SettingsProvider using InheritedWidget or Provider package. All settings must persist and apply globally in real-time.

// lib/providers/settings_provider.dart class SettingsProvider extends ChangeNotifier { bool _highContrast = false; double _textScale = 1.0; double _ttsSpeed = 0.5; String _defaultLevel = 'teen'; Future<void> loadFromPrefs() async { ... } Future<void> setHighContrast(bool val) async { _highContrast = val; await _prefs.setBool('high_contrast', val); notifyListeners(); // rebuilds entire app } }
6
Add Shake-to-Invoke MARK

Use the sensors_plus package to detect shake gestures globally. Invoke MARK from any screen without needing to find the FAB.

// In main.dart — wrap entire app class ShakeDetector extends StatefulWidget { // Use accelerometerEvents stream // Detect magnitude > threshold (2.7g) // Debounce 1 second between invocations // Call NarratorService + VoiceService globally }
